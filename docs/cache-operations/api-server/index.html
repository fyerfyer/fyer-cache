API Server

FyerCache 提供了一个内置 HTTP API 服务器，可以通过 HTTP 请求监控、管理和控制缓存实例。API 服务器支持缓存统计信息查询、手动失效缓存项、扩缩容操作以及健康检查等功能。

概览​

API 服务器是为管理和监控 FyerCache 实例而设计的 HTTP 接口。它提供了以下主要功能:

  • 统计信息 - 查询缓存使用情况和性能指标
  • 缓存失效 - 手动使特定缓存项失效
  • 集群管理 - 支持动态扩缩容操作（需要集成集群缓存提供者）
  • 健康检查 - 验证缓存服务的可用性

创建与配置 API 服务器​

创建 API 服务器需要提供一个基础缓存实例和可选的配置选项:

// 创建基础缓存
memCache := cache.NewMemoryCache()

// 创建 API 服务器
apiServer := api.NewAPIServer(
    memCache,
    api.WithBindAddress(":8080"),
    api.WithBasePath("/api"),
    api.WithTimeouts(10*time.Second, 10*time.Second, 60*time.Second),
)

// 启动服务器
err := apiServer.Start()
if err != nil {
    log.Fatalf("Failed to start API server: %v", err)
}

// 确保优雅关闭
defer apiServer.Stop()

配置集群功能（可选）​

如果要启用动态扩缩容功能，需要提供实现 ClusterCacheProvider 接口的缓存集群实例:

// 假设 clusterCache 实现了 ClusterCacheProvider 接口
apiServer.WithClusterCache(clusterCache)

API 端点​

API 服务器提供以下 HTTP 端点:

健康检查​

检查 API 服务器是否正常运行。

  • URL: /health
  • 方法: GET
  • 响应: 200 OK 和文本 "OK"
curl http://localhost:8080/health

获取缓存统计信息​

获取缓存的当前状态和性能统计数据。

  • URL: /api/stats
  • 方法: GET
  • 响应: 200 OK 和 JSON 格式的统计数据

响应示例:

{
  "item_count": 1250,
  "memory_usage": 2048576,
  "hit_rate": 0.85,
  "miss_rate": 0.15,
  "node_count": 3,
  "cluster_info": {
    "nodes": ["node1", "node2", "node3"]
  }
}

可返回的统计信息包括:

  • item_count: 缓存中的项目数量
  • memory_usage: 内存使用量（字节）
  • hit_rate: 缓存命中率
  • miss_rate: 缓存未命中率
  • node_count: 集群节点数量（如果是集群模式）
  • cluster_info: 集群相关信息（如果是集群模式）

失效缓存​

主动使某个缓存键失效。

  • URL: /api/invalidate?key=<cache_key>
  • 方法: POST
  • 响应: 200 OK 和操作状态
curl -X POST "http://localhost:8080/api/invalidate?key=user:1001"

响应示例:

{
  "success": true,
  "message": "Cache key 'user:1001' invalidated successfully",
  "data": {
    "key": "user:1001"
  }
}

扩缩容集群​

动态调整集群中的节点数量（仅在集成集群缓存提供者后可用）。

  • URL: /api/scale?nodes=<target_node_count>
  • 方法: POST
  • 响应: 200 OK 和操作结果
curl -X POST "http://localhost:8080/api/scale?nodes=5"

响应示例:

{
  "success": true,
  "message": "Cluster scaled successfully",
  "data": {
    "previous_node_count": 3,
    "current_node_count": 5,
    "added_nodes": ["node4", "node5"],
    "removed_nodes": []
  }
}

配置选项​

创建 API 服务器时可以使用以下配置选项:

WithBindAddress​

设置服务器绑定地址，格式为 "host:port"。

api.WithBindAddress(":8080") // 监听所有接口的 8080 端口

默认值: :8080

WithBasePath​

设置 API 端点的基础路径前缀。

api.WithBasePath("/cache/api")

默认值: /api

WithTimeouts​

设置 HTTP 服务器的超时参数。

api.WithTimeouts(
    5*time.Second,  // 读取超时
    10*time.Second, // 写入超时
    30*time.Second, // 空闲超时
)

也可以单独设置:

api.WithReadTimeout(5*time.Second)
api.WithWriteTimeout(10*time.Second)

默认值:

  • 读取超时: 10 秒
  • 写入超时: 10 秒
  • 空闲超时: 60 秒

API 服务器方法​

Start​

启动 API 服务器，开始监听请求。

err := apiServer.Start()

Stop​

停止 API 服务器，优雅关闭连接。

err := apiServer.Stop()

IsRunning​

检查 API 服务器是否正在运行。

if apiServer.IsRunning() {
    // 服务器正在运行
}

Address​

获取 API 服务器的绑定地址。

addr := apiServer.Address()
log.Printf("API server running at %s", addr)

集成示例​

基本集成​

package main

import (
    "log"
    "time"
    
    "github.com/fyerfyer/fyer-cache/cache"
    "github.com/fyerfyer/fyer-cache/cache/api"
)

func main() {
    // 创建内存缓存
    memCache := cache.NewMemoryCache()
    
    // 创建 API 服务器
    apiServer := api.NewAPIServer(
        memCache,
        api.WithBindAddress(":8081"),
        api.WithBasePath("/cache/api"),
    )
    
    // 启动服务器
    if err := apiServer.Start(); err != nil {
        log.Fatalf("Failed to start API server: %v", err)
    }
    
    log.Printf("API server running at %s", apiServer.Address())
    
    // 应用主循环...
    
    // 程序退出时优雅关闭
    if err := apiServer.Stop(); err != nil {
        log.Printf("Error stopping API server: %v", err)
    }
}

集群模式集成​

package main

import (
    "log"
    "time"
    
    "github.com/fyerfyer/fyer-cache/cache"
    "github.com/fyerfyer/fyer-cache/cache/api"
    "github.com/fyerfyer/fyer-cache/cache/cluster/integration"
)

func main() {
    // 创建集群缓存
    clusterCache, err := integration.NewShardedClusterCache(
        cache.NewMemoryCache(),
        "node1",
        "localhost:7946",
        // 其他配置...
    )
    if err != nil {
        log.Fatalf("Failed to create cluster cache: %v", err)
    }
    
    // 启动集群节点
    if err := clusterCache.Start(); err != nil {
        log.Fatalf("Failed to start cluster: %v", err)
    }
    defer clusterCache.Stop()
    
    // 创建 API 服务器
    apiServer := api.NewAPIServer(
        clusterCache,
        api.WithBindAddress(":8081"),
    )
    
    // 设置集群提供者以启用扩缩容功能
    apiServer.WithClusterCache(clusterCache)
    
    // 启动 API 服务器
    if err := apiServer.Start(); err != nil {
        log.Fatalf("Failed to start API server: %v", err)
    }
    defer apiServer.Stop()
    
    // 应用主循环...
}

安全考量​

API 服务器未内置认证和授权机制，因此在生产环境中部署时请考虑以下安全措施:

  1. 网络隔离 - 将 API 服务器绑定到内部网络接口
  2. 反向代理 - 使用 Nginx 或 Traefik 等反向代理添加认证层
  3. API 密钥 - 实现自定义中间件以验证请求中的 API 密钥
  4. TLS 加密 - 配置 HTTPS 保护传输层安全

监控与调试​

API 服务器是监控 FyerCache 实例的工具。使用 /api/stats 端点可以:

  1. 实现自动监控脚本，定期检查缓存健康状态
  2. 配置监控系统如 Prometheus 抓取指标
  3. 创建可视化仪表板展示缓存性能
  4. 设置基于阈值的告警（如内存使用量过高）

定制 API 服务器​

对于更高级的需求，可以扩展 API 服务器的功能:

  1. 在自定义 HTTP 处理器中包装 API 服务器
  2. 添加日志记录中间件跟踪请求
  3. 实现额外的 API 端点满足特定需求
  4. 添加自定义指标收集功能
