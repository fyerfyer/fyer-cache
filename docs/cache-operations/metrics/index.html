Metrics Monitoring

FyerCache 内置了强大的指标监控系统，帮助您追踪缓存性能、资源使用和操作统计。本文档详细介绍了 FyerCache 的监控架构、可用指标以及配置方法。

监控架构​

FyerCache 的监控系统基于以下组件设计：

┌───────────────┐      ┌───────────────┐      ┌───────────────┐
│  MonitoredCache  │     │   Collector   │     │    Exporter   │
│  (装饰器模式)  │────▶│ (指标收集)  │────▶│ (指标导出)  │
└───────────────┘      └───────────────┘      └───────────────┘
                                                      │
                                                      ▼
                                             ┌───────────────┐
                                             │ Metrics Server │
                                             │   (HTTP服务)  │
                                             └───────────────┘

核心组件​

  1. MonitoredCache：装饰器模式包装原始缓存，拦截操作并收集指标
  2. Collector：收集和聚合指标数据
  3. Exporter：将收集的指标转换为标准格式（如Prometheus）
  4. Metrics Server：通过HTTP端点暴露指标，用于监控系统抓取

可用指标​

FyerCache 默认收集以下核心指标：

缓存操作指标​

指标名称类型描述标签
fyercache_cache_hit_count计数器缓存命中总数-
fyercache_cache_miss_count计数器缓存未命中总数-
fyercache_cache_request_count计数器缓存请求总数operation
fyercache_cache_operation_latency_seconds直方图操作延迟分布operation

资源使用指标​

指标名称类型描述标签
fyercache_cache_memory_usage_bytes仪表盘缓存使用的内存字节数-
fyercache_cache_item_count仪表盘缓存中的项目数量-

计算指标​

指标名称计算方式描述
命中率hit_count / (hit_count + miss_count)缓存命中的比率
未命中率miss_count / (hit_count + miss_count)缓存未命中的比率

启用监控​

要为缓存启用指标监控，只需将原始缓存包装在 MonitoredCache 中：

// 创建基础缓存
originalCache := cache.NewMemoryCache()

// 添加监控装饰器
monitoredCache := metrics.NewMonitoredCache(
    originalCache,
    // 配置选项
    metrics.WithMetricsServer(":8080"),
    metrics.WithLabels(map[string]string{
        "service": "user-service",
        "env":     "production",
    }),
)

配置选项​

FyerCache 提供多种监控配置选项：

服务器配置​

// 指定监控服务器的地址和端口
metrics.WithMetricsServer(":8080")

// 自定义指标路径（默认为 "/metrics"）
metrics.WithMetricsPath("/custom/metrics")

收集配置​

// 设置指标收集间隔（默认为 1分钟）
metrics.WithCollectInterval(30 * time.Second)

// 设置内存使用量统计更新间隔（默认为 1分钟）
metrics.WithMemoryUpdateInterval(30 * time.Second)

// 启用或禁用内存统计（默认为启用）
metrics.WithEnableMemStats(true)

指标标签和命名空间​

// 设置自定义标签，将添加到所有指标
metrics.WithLabels(map[string]string{
    "service": "user-service",
    "region":  "us-west",
    "env":     "production",
})

// 设置指标命名空间（默认为 "fyercache"）
metrics.WithNamespace("myapp")

// 设置指标子系统名称（默认为 "cache"）
metrics.WithSubsystem("redis")

指标收集机制​

FyerCache 使用两种方式收集指标：

  1. 操作拦截：MonitoredCache 装饰器拦截所有缓存操作（Get、Set、Del），记录命中/未命中、请求数和延迟。
// Get 获取缓存键值，同时收集指标
func (mc *MonitoredCache) Get(ctx context.Context, key string) (any, error) {
    // 记录请求计数
    mc.collector.IncRequests("get")
    
    // 记录开始时间，用于计算延迟
    start := time.Now()
    
    // 调用底层缓存实现
    value, err := mc.cache.Get(ctx, key)
    
    // 记录延迟
    mc.collector.ObserveLatency("get", time.Since(start))
    
    // 记录命中/未命中
    if err != nil {
        if errors.Is(err, ferr.ErrKeyNotFound) {
            mc.collector.IncMisses()
        }
    } else {
        mc.collector.IncHits()
    }
    
    return value, err
}
  1. 周期性收集：后台任务定期收集资源指标（内存使用、条目数）。
// 周期性收集指标
func (mc *MonitoredCache) collectMetrics() {
    ticker := time.NewTicker(mc.collectInterval)
    defer ticker.Stop()
    
    for {
        select {
        case <-mc.stopCh:
            return
        case <-ticker.C:
            // 获取项目数量
            itemCount := mc.getItemCount()
            mc.collector.SetItemCount(itemCount)
        }
    }
}

实现细节​

Prometheus 适配​

FyerCache 默认使用 Prometheus 作为底层指标系统，实现了 PrometheusCollector 和 PrometheusExporter：

// 创建内部 Prometheus 注册表
registry := prometheus.NewRegistry()

// 创建计数器
hitCounter := prometheus.NewCounter(
    prometheus.CounterOpts{
        Namespace: namespace,
        Subsystem: subsystem,
        Name:      MetricHitCount,
        Help:      HelpHitCount,
    },
)
registry.MustRegister(hitCounter)

// 创建直方图
latencyHistogram := prometheus.NewHistogramVec(
    prometheus.HistogramOpts{
        Namespace: namespace,
        Subsystem: subsystem,
        Name:      MetricLatency,
        Help:      HelpLatency,
        Buckets:   []float64{.001, .005, .01, .025, .05, .1, .25, .5, 1, 2.5, 5, 10},
    },
    []string{"operation"},
)
registry.MustRegister(latencyHistogram)

HTTP 服务器​

指标通过 HTTP 服务器暴露，使用 Prometheus HTTP 处理器：

// 创建HTTP服务器处理器
func (pe *PrometheusExporter) Export(w http.ResponseWriter) {
    // 创建Prometheus HTTP处理器
    h := promhttp.HandlerFor(pe.registry, promhttp.HandlerOpts{})
    
    // 处理请求
    req, _ := http.NewRequest("GET", "/metrics", nil)
    h.ServeHTTP(w, req)
}

使用示例​

基本监控设置​

package main

import (
	"github.com/fyerfyer/fyer-cache/cache"
	"github.com/fyerfyer/fyer-cache/cache/metrics"
	"time"
)

func main() {
	// 创建基础缓存
	memCache := cache.NewMemoryCache()
	
	// 创建监控包装器
	monitoredCache := metrics.NewMonitoredCache(
		memCache,
		metrics.WithMetricsServer(":8080"),
		metrics.WithCollectInterval(1*time.Minute),
		metrics.WithLabels(map[string]string{
			"service": "user-service",
		}),
	)
	
	// 正常使用缓存，所有操作都会自动收集指标
	// ...
	
	// 应用关闭时停止监控
	defer monitoredCache.Stop()
}

与 Prometheus 和 Grafana 集成​

  1. 配置 Prometheus
# prometheus.yml
scrape_configs:
  - job_name: 'fyercache'
    scrape_interval: 15s
    static_configs:
      - targets: ['localhost:8080']
  1. 设置 Grafana 仪表板

创建 Grafana 仪表板，添加以下面板：

  • 命中率：rate(fyercache_cache_hit_count[5m]) / (rate(fyercache_cache_hit_count[5m]) + rate(fyercache_cache_miss_count[5m]))
  • 每秒请求数：sum(rate(fyercache_cache_request_count[5m])) by (operation)
  • 操作延迟 P95：histogram_quantile(0.95, sum(rate(fyercache_cache_operation_latency_seconds_bucket[5m])) by (le, operation))
  • 内存使用量：fyercache_cache_memory_usage_bytes
  • 缓存条目数：fyercache_cache_item_count
