Distributed Cache Examples

远程缓存客户端​

remote_cache_client.go 示例演示了如何创建和使用远程缓存客户端，连接到远程缓存服务器：

  • 创建基本的远程缓存客户端
  • 设置和获取缓存值
  • 配置超时和重试策略
  • 处理连接错误和异常情况

关键代码段：

// 创建基本的远程缓存客户端
remoteCache := distributed.NewRemoteCache(nodeID, nodeAddress)

// 设置缓存项，1分钟过期
err := remoteCache.Set(ctx, "user:1001", "John Doe", 1*time.Minute)

// 获取缓存项
val, err := remoteCache.Get(ctx, "user:1001")

// 使用自定义配置创建客户端
customCache := distributed.NewRemoteCache(
    "remote-node-2",
    "localhost:8081",
    distributed.WithTimeout(500*time.Millisecond),  // 较短的超时时间
    distributed.WithRetry(5, 200*time.Millisecond), // 更多重试次数和间隔
)

这个示例展示了如何创建远程缓存客户端并执行基本操作，是分布式缓存使用的入门示例。

缓存服务器​

server.go 示例演示了如何实现一个简单的缓存服务器，提供 HTTP API 接口：

  • 实现基本的 HTTP 缓存服务器
  • 提供 GET、SET、DELETE 缓存操作的 API 端点
  • 处理 JSON 请求和响应
  • 记录详细的请求日志
  • 提供健康检查端点

关键代码段：

// 设置 HTTP 路由
mux.HandleFunc("/cache/get", func(w http.ResponseWriter, r *http.Request) {
    // 解析请求体中的键
    var req CacheRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, "Invalid request body", http.StatusBadRequest)
        return
    }

    // 从缓存获取值
    cacheLock.Lock()
    value, exists := memCache[req.Key]
    cacheLock.Unlock()

    // 返回结果
    if !exists {
        writeJSONResponse(w, CacheResponse{
            Success: false,
            Message: "Key not found",
        })
        return
    }

    writeJSONResponse(w, CacheResponse{
        Success: true,
        Value:   value,
    })
})

此示例实现了与远程缓存客户端配套的服务器端，用于在实际环境中快速搭建分布式缓存服务。

节点分片缓存​

node_sharded_cache.go 示例展示了如何使用基于节点的分片缓存进行数据分布：

  • 创建基于节点的分片缓存
  • 添加多个缓存节点并设置权重
  • 存取分布在不同节点的数据
  • 演示一致性哈希的特性（添加/删除节点时最小化数据重新映射）
  • 测试节点权重对数据分布的影响

关键代码段：

// 创建基于节点的分片缓存
shardedCache := cache.NewNodeShardedCache()

// 创建后端存储节点
node1 := cache.NewMemoryCache()
node2 := cache.NewMemoryCache()
node3 := cache.NewMemoryCache()

// 添加节点到分片缓存（参数: nodeID, cache实例, 权重）
err := shardedCache.AddNode("node1", node1, 1)
err = shardedCache.AddNode("node2", node2, 1)
err = shardedCache.AddNode("node3", node3, 1)

// 设置和获取数据，分片缓存自动处理路由
shardedCache.Set(ctx, key, value, 10*time.Minute)
value, err := shardedCache.Get(ctx, key)

该示例还演示了添加和移除节点时一致性哈希的特性，以及如何使用不同权重影响数据分布。

运行示例​

要运行这些示例，您可以使用以下步骤：

1. 运行缓存服务器​

# 启动缓存服务器
go run example/distributed/server/server.go

服务器将在 :8080 端口启动，并提供如下 HTTP 端点：

  • POST /cache/get - 获取缓存项
  • POST /cache/set - 设置缓存项
  • POST /cache/delete - 删除缓存项
  • GET /health - 健康检查

2. 运行远程缓存客户端​

# 在另一个终端运行缓存客户端
go run example/distributed/remote_client/remote_cache_client.go

客户端将连接到 localhost:8080 并执行一系列缓存操作。

3. 运行节点分片缓存示例​

# 运行节点分片缓存示例
go run example/distributed/node_sharded/node_sharded_cache.go

这将在单个进程中演示节点分片和一致性哈希的特性。
