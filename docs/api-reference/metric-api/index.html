Metric API

核心组件​

FyerCache 的监控系统由以下核心组件组成：

Collector（收集器）​

Collector 接口负责收集和聚合指标数据：

type Collector interface {
    // 更新缓存命中/未命中计数
    IncHits()
    IncMisses()

    // 记录请求计数
    IncRequests(operation string)

    // 记录操作延迟
    ObserveLatency(operation string, duration time.Duration)

    // 设置内存用量和条目数
    SetMemoryUsage(bytes int64)
    SetItemCount(count int64)

    // 获取当前命中率
    HitRate() float64
}

默认实现：PrometheusCollector

Exporter（导出器）​

Exporter 接口负责将收集的指标转换为可消费格式（如 Prometheus）：

type Exporter interface {
    // 导出收集到的指标
    Export(w http.ResponseWriter)

    // 注册新的指标
    RegisterMetric(name, help string, typ MetricType, labels ...string)
}

默认实现：PrometheusExporter

Provider（提供者）​

Provider 接口结合了收集器和导出器的功能：

type Provider interface {
    Collector
    Exporter

    // 启动指标暴露服务
    Start() error

    // 停止指标暴露服务
    Stop() error
}

Server（服务器）​

Server 提供 HTTP 服务器，用于暴露指标端点：

// Server 提供HTTP服务器，暴露指标端点
type Server struct {
    exporter Exporter
    config   *MetricsConfig
    server   *http.Server
    running  bool
    mu       sync.RWMutex
}

MonitoredCache（监控装饰器）​

MonitoredCache 是一个缓存装饰器，拦截缓存操作并收集指标：

// MonitoredCache 用于包装缓存实例，添加监控功能
type MonitoredCache struct {
    cache             cache.Cache
    collector         Collector
    exporter          Exporter
    server            *Server
    namespace         string
    subsystem         string
    collectInterval   time.Duration
    memoryUpdateInterval time.Duration
    enableMemStats    bool
    stopCh            chan struct{}
    labels            map[string]string
    running           bool
    mu                sync.Mutex
}

可用指标​

FyerCache 默认提供以下指标：

指标名称类型描述标签
{namespace}_{subsystem}_hit_count计数器缓存命中总数自定义标签
{namespace}_{subsystem}_miss_count计数器缓存未命中总数自定义标签
{namespace}_{subsystem}_request_count计数器缓存请求总数operation, 自定义标签
{namespace}_{subsystem}_operation_latency_seconds直方图操作延迟分布（秒）operation, 自定义标签
{namespace}_{subsystem}_memory_usage_bytes仪表盘当前内存使用量（字节）自定义标签
{namespace}_{subsystem}_item_count仪表盘当前缓存项数量自定义标签

默认值：

  • namespace = "fyercache"
  • subsystem = "cache"

创建监控缓存​

要为现有缓存添加监控功能，请使用 MonitoredCache 装饰器：

// 创建基础缓存
originalCache := cache.NewMemoryCache()

// 添加监控装饰器
monitoredCache := metrics.NewMonitoredCache(
    originalCache,
    metrics.WithMetricsServer(":8080"),
    metrics.WithLabels(map[string]string{
        "service": "user-service",
        "env":     "production",
    }),
)

// 使用 monitoredCache 替代 originalCache
// 缓存操作将自动收集指标

配置选项​

服务器配置​

// 指定指标 HTTP 服务器地址
metrics.WithMetricsServer(":8080")

// 自定义指标路径（默认为 "/metrics"）
metrics.WithMetricsPath("/custom/metrics")

收集配置​

// 设置指标收集间隔（默认为 1 分钟）
metrics.WithCollectInterval(30 * time.Second)

// 设置内存使用量统计更新间隔（默认为 1 分钟）
metrics.WithMemoryUpdateInterval(30 * time.Second)

// 启用或禁用内存统计（默认为启用）
metrics.WithEnableMemStats(true)

指标命名和标签​

// 设置自定义标签，将添加到所有指标
metrics.WithLabels(map[string]string{
    "service": "user-service",
    "region":  "us-west",
    "env":     "production",
})

// 设置指标命名空间（默认为 "fyercache"）
metrics.WithNamespace("myapp")

// 设置指标子系统名称（默认为 "cache"）
metrics.WithSubsystem("redis")

HTTP 端点​

启用指标服务器后，以下 HTTP 端点可用：

指标端点​

  • URL: /metrics（可通过 WithMetricsPath 配置）
  • 方法: GET
  • 描述: 以 Prometheus 格式返回所有收集的指标
  • 响应: Prometheus 文本格式的指标数据

示例响应：

# HELP fyercache_cache_hit_count Total number of cache hits
# TYPE fyercache_cache_hit_count counter
fyercache_cache_hit_count{env="production",service="user-service"} 1285

# HELP fyercache_cache_miss_count Total number of cache misses
# TYPE fyercache_cache_miss_count counter
fyercache_cache_miss_count{env="production",service="user-service"} 348

# HELP fyercache_cache_operation_latency_seconds Latency distribution of cache operations in seconds
# TYPE fyercache_cache_operation_latency_seconds histogram
fyercache_cache_operation_latency_seconds_bucket{env="production",operation="get",service="user-service",le="0.001"} 762
...

# HELP fyercache_cache_item_count Current number of items in the cache
# TYPE fyercache_cache_item_count gauge
fyercache_cache_item_count{env="production",service="user-service"} 2548

健康检查端点​

  • URL: /health
  • 方法: GET
  • 描述: 检查指标服务器是否正常运行
  • 响应: 200 OK 状态码和 "OK" 文本

注册自定义指标​

您可以注册自定义指标以跟踪特定应用指标：

// 获取导出器
exporter := monitoredCache.exporter

// 注册自定义计数器
exporter.RegisterMetric(
    "cache_eviction_count",  // 指标名称
    "Number of items evicted from cache",  // 指标描述
    metrics.TypeCounter,  // 指标类型
    "reason"  // 可选标签
)

// 注册自定义仪表盘
exporter.RegisterMetric(
    "cache_max_memory_limit",
    "Maximum memory limit for cache in bytes",
    metrics.TypeGauge
)

访问指标计算值​

MonitoredCache 提供了访问计算指标的方法：

// 获取缓存命中率
hitRate := monitoredCache.HitRate()
fmt.Printf("Current cache hit rate: %.2f%%\n", hitRate * 100)

// 如果缓存实现了 MemoryStatsProvider 接口，可以获取内存使用情况
if statsProvider, ok := monitoredCache.(cache.MemoryStatsProvider); ok {
    memoryUsage := statsProvider.MemoryUsage()
    itemCount := statsProvider.ItemCount()
    fmt.Printf("Memory usage: %d bytes, Items: %d\n", memoryUsage, itemCount)
}

指标收集机制​

FyerCache 使用两种方式收集指标：

  1. 操作拦截 - 装饰器模式拦截缓存操作：
// MonitoredCache.Get 示例
func (mc *MonitoredCache) Get(ctx context.Context, key string) (any, error) {
    // 记录请求计数
    mc.collector.IncRequests("get")
    
    // 记录开始时间
    start := time.Now()
    
    // 调用底层缓存
    value, err := mc.cache.Get(ctx, key)
    
    // 记录延迟
    mc.collector.ObserveLatency("get", time.Since(start))
    
    // 记录命中/未命中
    if err != nil {
        if errors.Is(err, ferr.ErrKeyNotFound) {
            mc.collector.IncMisses()
        }
    } else {
        mc.collector.IncHits()
    }
    
    return value, err
}
  1. 周期性收集 - 定期收集资源指标：
// 周期性收集内存使用和条目数
func (mc *MonitoredCache) collectMemStats() {
    ticker := time.NewTicker(mc.memoryUpdateInterval)
    defer ticker.Stop()
    
    for {
        select {
        case <-mc.stopCh:
            return
        case <-ticker.C:
            if mc.enableMemStats {
                // 如果缓存实现了 MemoryStatsProvider 接口
                if provider, ok := mc.cache.(cache.MemoryStatsProvider); ok {
                    memoryUsage := provider.MemoryUsage()
                    mc.collector.SetMemoryUsage(memoryUsage)
                }
            }
            
            // 收集项目数量
            itemCount := mc.getItemCount()
            mc.collector.SetItemCount(itemCount)
        }
    }
}

停止监控​

当应用退出时，应正确停止监控以释放资源：

// 停止监控服务器和收集器
err := monitoredCache.Stop()
if err != nil {
    log.Printf("Error stopping metrics: %v", err)
}

实现自定义收集器/导出器​

如果需要支持其他指标系统（如 StatsD、DataDog 等），可以通过实现 Collector 和 Exporter 接口创建自定义实现：

// 实现 Collector 接口的自定义收集器
type MyCustomCollector struct {
    // 内部字段...
}

// 实现所需方法...

// 实现 Exporter 接口的自定义导出器
type MyCustomExporter struct {
    collector *MyCustomCollector
    // 其他字段...
}

// 实现所需方法...

// 创建自定义监控缓存
customMonitored := metrics.NewCustomMonitoredCache(
    originalCache,
    myCollector,
    myExporter,
    // 其他选项...
)

与 Prometheus 集成​

FyerCache 的默认指标实现与 Prometheus 兼容。在 Prometheus 配置中添加以下抓取配置：

scrape_configs:
  - job_name: 'fyercache'
    scrape_interval: 15s
    static_configs:
      - targets: ['localhost:8080']
