HTTP API

总览​

FyerCache 提供了三类 HTTP API:

  1. 管理 API - 用于监控和控制缓存系统
  2. 缓存操作 API - 用于基本的缓存读写操作
  3. 复制同步 API - 用于节点间的数据同步

所有 API 都使用 JSON 格式进行数据交换。除非特别说明，标准响应格式为:

{
  "success": true,
  "message": "操作成功或错误描述",
  "data": { }  // 操作相关的数据
}

管理 API​

管理 API 由 API 服务器组件提供，默认绑定在 :8080 并使用 /api 基础路径。

获取缓存统计信息​

获取当前缓存的状态和性能统计数据。

  • URL: /api/stats
  • 方法: GET
  • 权限: 无需认证

响应:

{
  "item_count": 1250,
  "memory_usage": 2048576,
  "hit_rate": 0.85,
  "miss_rate": 0.15,
  "node_count": 3,
  "shard_count": 32,
  "cluster_info": {
    "nodes": ["node1", "node2", "node3"]
  }
}
字段类型描述
item_count整数缓存中的项目数量
memory_usage整数当前内存使用量（字节）
hit_rate浮点缓存命中率（0-1）
miss_rate浮点缓存未命中率（0-1）
node_count整数集群中节点数量
shard_count整数分片数量
cluster_info对象集群相关信息

使缓存键失效​

手动删除特定的缓存键。

  • URL: /api/invalidate?key={cache_key}
  • 方法: POST
  • 权限: 无需认证

参数:

参数类型必填描述
key字符串是要删除的缓存键

响应:

{
  "success": true,
  "message": "Cache key 'user:1001' invalidated successfully",
  "data": {
    "key": "user:1001"
  }
}

扩缩容集群​

动态调整集群中的节点数量（仅当集群缓存提供者可用时）。

  • URL: /api/scale?nodes={target_node_count}
  • 方法: POST
  • 权限: 无需认证

参数:

参数类型必填描述
nodes整数是目标节点数量

响应:

{
  "success": true,
  "message": "Cluster scaled successfully",
  "data": {
    "previous_node_count": 3,
    "current_node_count": 5,
    "added_nodes": ["node4", "node5"],
    "removed_nodes": []
  }
}

缓存操作 API​

缓存操作 API 由分布式缓存服务器组件提供，用于基本的缓存读写操作，默认绑定在 :8080。

获取缓存项​

从缓存中检索特定键的值。

  • URL: /cache/get
  • 方法: POST
  • 权限: 无需认证
  • 内容类型: application/json

请求体:

{
  "key": "user:1001"
}

响应:

{
  "success": true,
  "message": "",
  "value": {
    "name": "John Doe",
    "age": 30
  }
}

设置缓存项​

在缓存中存储键值对。

  • URL: /cache/set
  • 方法: POST
  • 权限: 无需认证
  • 内容类型: application/json

请求体:

{
  "key": "user:1001",
  "value": {
    "name": "John Doe",
    "age": 30
  },
  "expiration": 3600000000000
}

注意: expiration 字段以纳秒为单位表示过期时间。例如，3600000000000 表示 1 小时。

响应:

{
  "success": true,
  "message": "Value stored successfully"
}

删除缓存项​

从缓存中删除特定键。

  • URL: /cache/delete
  • 方法: DELETE 或 POST
  • 权限: 无需认证
  • 内容类型: application/json

请求体:

{
  "key": "user:1001"
}

响应:

{
  "success": true,
  "message": "Value deleted successfully"
}

复制同步 API​

复制同步 API 用于主从节点间的数据同步，通常不需要直接调用，由系统内部使用。

同步请求​

请求数据同步操作。

  • URL: /replication/sync
  • 方法: POST
  • 权限: 无需认证
  • 内容类型: application/json

请求体:

{
  "leader_id": "node1",
  "last_log_index": 1042,
  "last_log_term": 3,
  "full_sync": false
}

响应:

{
  "success": true,
  "entries": [
    {
      "index": 1043,
      "term": 3,
      "command": "Set",
      "key": "product:10234",
      "value": "eyJuYW1lIjoiRGVza3RvcCBQQyIsInByaWNlIjoxMjk5Ljk5fQ==",
      "expiration": 3600000000000,
      "timestamp": "2025-03-19T10:15:30Z"
    }
  ],
  "next_index": 1044
}

健康检查 API​

所有组件都提供健康检查端点，用于监控系统健康状态。

API 服务器健康检查​

  • URL: /health
  • 方法: GET
  • 权限: 无需认证

响应: 200 OK 和文本 "OK"

节点健康检查​

  • URL: /health
  • 方法: GET
  • 权限: 无需认证

响应: 200 OK 和文本 "OK"

复制同步健康检查​

  • URL: /replication/health
  • 方法: GET
  • 权限: 无需认证

响应: 200 OK 和文本 "OK"

错误处理​

当 API 调用失败时，响应将包含相关错误信息：

{
  "success": false,
  "message": "错误描述",
  "error": "错误类型或代码"
}

常见 HTTP 状态码：

状态码描述
200成功
400请求格式错误或参数无效
404资源或端点不存在
409操作冲突
500服务器内部错误

实际使用示例​

以下是使用 curl 工具访问 FyerCache API 的示例：

获取缓存统计信息​

curl -X GET http://localhost:8080/api/stats

设置缓存项​

curl -X POST http://localhost:8080/cache/set \
  -H "Content-Type: application/json" \
  -d '{"key":"user:1001","value":{"name":"John Doe","email":"john@example.com"},"expiration":3600000000000}'

获取缓存项​

curl -X POST http://localhost:8080/cache/get \
  -H "Content-Type: application/json" \
  -d '{"key":"user:1001"}'

删除缓存项​

curl -X POST http://localhost:8080/cache/delete \
  -H "Content-Type: application/json" \
  -d '{"key":"user:1001"}'

手动使缓存键失效​

curl -X POST "http://localhost:8080/api/invalidate?key=user:1001"

扩容集群到 5 个节点​

curl -X POST "http://localhost:8080/api/scale?nodes=5"

检查系统健康状态​

curl -X GET http://localhost:8080/health

API 安全性考量​

默认情况下，FyerCache 的 HTTP API 没有内置认证机制。在生产环境中，建议采取以下措施增强安全性：

  1. 网络隔离 - 将 API 服务器限制在内部网络或专用网段
  2. 反向代理 - 使用 Nginx 或其他反向代理添加认证层
  3. API 密钥 - 实现自定义中间件验证 API 密钥
  4. HTTPS - 配置 TLS 加密保护传输层安全

自定义端口和路径​

API 服务器支持自定义绑定地址和基础路径：

apiServer := api.NewAPIServer(
    cache,
    api.WithBindAddress(":9000"),         // 自定义端口
    api.WithBasePath("/custom/api"),      // 自定义基础路径
)

使用自定义配置后，API 端点会相应变化，例如统计信息端点将变为 /custom/api/stats。
