Cache Interface & Method

核心接口 Cache​

所有 FyerCache 实现都基于 cache.Cache 接口，定义了三个核心方法：

type Cache interface {
    Get(ctx context.Context, key string) (any, error)
    Set(ctx context.Context, key string, val any, expiration time.Duration) error
    Del(ctx context.Context, key string) error
}

Get 方法​

从缓存中检索与指定键关联的值。

签名：

Get(ctx context.Context, key string) (any, error)

参数：

  • ctx context.Context：上下文对象，用于传递请求范围的值、取消信号和截止时间
  • key string：要检索的缓存项的键

返回值：

  • any：存储的值，可以是任何类型
  • error：操作过程中发生的任何错误

错误：

  • 如果键不存在，返回 ferr.ErrKeyNotFound
  • 在分布式环境中，可能返回网络或远程节点错误

示例：

value, err := cache.Get(context.Background(), "user:1001")
if err != nil {
    if errors.Is(err, ferr.ErrKeyNotFound) {
        // 键不存在
    } else {
        // 其他错误
    }
    return
}

// 使用类型断言转换为预期类型
userInfo, ok := value.(UserInfo)
if !ok {
    // 类型不匹配
}

Set 方法​

将值与指定键关联存储在缓存中。

签名：

Set(ctx context.Context, key string, val any, expiration time.Duration) error

参数：

  • ctx context.Context：上下文对象
  • key string：缓存项的键
  • val any：要存储的值，可以是任何类型
  • expiration time.Duration：过期时间，从调用 Set 开始计算
    • 值为 0 表示永不过期
    • 负值通常视为立即过期或使用默认过期时间（取决于实现）

返回值：

  • error：操作过程中发生的任何错误

示例：

user := UserInfo{
    ID:   1001,
    Name: "John Doe",
    Age:  30,
}

// 存储用户信息，10分钟后过期
err := cache.Set(context.Background(), "user:1001", user, 10*time.Minute)
if err != nil {
    // 处理错误
}

// 存储不过期的值
err = cache.Set(context.Background(), "constants:pi", 3.14159, 0)

Del 方法​

从缓存中删除指定键的项。

签名：

Del(ctx context.Context, key string) error

参数：

  • ctx context.Context：上下文对象
  • key string：要删除的缓存项的键

返回值：

  • error：操作过程中发生的任何错误

注意：

  • 如果键不存在，大多数实现不会返回错误
  • 在需要确保键存在的场景下，应首先使用 Get 方法检查

示例：

err := cache.Del(context.Background(), "user:1001")
if err != nil {
    // 处理错误
}

扩展接口​

DistributedCache 接口​

distributed.DistributedCache 接口扩展了基本的 Cache 接口，增加了对分布式环境的支持：

type DistributedCache interface {
    // 核心缓存操作
    Get(ctx context.Context, key string) (any, error)
    Set(ctx context.Context, key string, value any, expiration time.Duration) error
    Del(ctx context.Context, key string) error

    // 本地缓存操作
    GetLocal(ctx context.Context, key string) (any, error)
    SetLocal(ctx context.Context, key string, value any, expiration time.Duration) error
    DelLocal(ctx context.Context, key string) error

    // 节点管理
    NodeID() string
    NodeAddress() string
}

本地操作方法​

本地方法直接操作当前节点的数据，绕过分布式路由：

  • GetLocal - 从本地缓存获取值，不考虑分片或路由
  • SetLocal - 在本地缓存设置值，不传播到其他节点
  • DelLocal - 删除本地缓存中的值，不影响其他节点

节点管理方法​

  • NodeID() - 返回当前节点的唯一标识
  • NodeAddress() - 返回当前节点的网络地址

ShardedCache 接口​

cache.ShardedCache 接口扩展了基本的 Cache 接口，提供了分片缓存的节点管理功能：

type ShardedCache interface {
    Cache

    // AddNode 添加缓存节点
    AddNode(nodeID string, cache Cache, weight int) error

    // RemoveNode 移除缓存节点
    RemoveNode(nodeID string) error

    // GetNodeIDs 获取所有节点ID
    GetNodeIDs() []string

    // GetNodeCount 获取节点数量
    GetNodeCount() int
}
  • AddNode - 添加一个新的缓存节点到分片集合中
  • RemoveNode - 从分片集合中移除一个缓存节点
  • GetNodeIDs - 获取所有缓存节点的ID列表
  • GetNodeCount - 获取缓存节点的总数

主要实现​

FyerCache 提供了多种缓存接口的具体实现：

MemoryCache​

内存中的本地缓存实现，适用于单节点应用。

cache := cache.NewMemoryCache()

配置选项：

  • WithEvictionCallback - 设置键被淘汰时的回调函数
  • WithCleanupInterval - 设置清理过期键的间隔
  • WithShardCount - 设置内部分片数量以提高并发性能
  • WithAsyncCleanup - 启用异步清理

NodeShardedCache​

基于一致性哈希的分片缓存，将键映射到不同的缓存节点。

shardedCache := cache.NewNodeShardedCache(
    cache.WithNodeHashReplicas(100),  // 每个物理节点的虚拟节点数
    cache.WithNodeReplicaFactor(2),   // 数据副本数量 (容错)
)

// 添加节点
shardedCache.AddNode("node1", node1Cache, 1)  // 权重为1
shardedCache.AddNode("node2", node2Cache, 2)  // 权重为2

ClusterCache​

结合集群管理和分片功能的缓存实现，用于动态集群。

clusterCache, err := cluster.NewClusterCache(
    localCache, 
    "node-1", 
    ":7946",
    cluster.WithGossipInterval(500 * time.Millisecond),
)

// 加入集群
clusterCache.Join("192.168.1.100:7946")

RemoteCache​

连接到远程缓存节点的客户端实现。

remoteCache := distributed.NewRemoteCache(
    "remote-node-1", 
    "cache.example.com:8080",
    distributed.WithTimeout(500 * time.Millisecond),
    distributed.WithRetry(3, 100 * time.Millisecond),
)

缓存一致性策略​

FyerCache 提供了不同的缓存一致性策略实现，用于协调缓存和后端存储之间的数据一致性：

CacheAsideStrategy​

"旁路缓存"策略实现。首先查询缓存，缓存未命中时从数据源读取。

strategy := consistency.NewCacheAsideStrategy(
    cacheImpl, 
    dataSource,
    consistency.WithCacheAsideTTL(10 * time.Minute),
    consistency.WithCacheAsideWriteOnMiss(true),
)

// 使用策略
value, err := strategy.Get(ctx, key)

// 设置值
err = strategy.Set(ctx, key, value, expiration)

WriteThroughStrategy​

"直写"策略实现。写操作同时更新缓存和数据源。

strategy := consistency.NewWriteThroughStrategy(
    cacheImpl, 
    dataSource,
    consistency.WithWriteThroughTTL(10 * time.Minute),
    consistency.WithDelayedWrite(true, 100 * time.Millisecond),
)

MQNotifierStrategy​

基于消息队列的缓存一致性策略。数据源变更时通过消息队列通知缓存失效。

strategy := consistency.NewMQNotifierStrategy(
    cacheImpl, 
    dataSource,
    mqPublisher,
    consistency.WithMQTopic("cache:invalidation"),
    consistency.WithMQNotifierTTL(10 * time.Minute),
)
