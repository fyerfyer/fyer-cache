Cluster Formation & Membership Management

FyerCache 支持动态集群形成和成员管理，通过去中心化的设计实现高可用性和可扩展性。

集群架构概述​

FyerCache 的集群管理基于一个去中心化的架构，核心包括以下组件：

核心组件​

  1. 节点 (Node) - 集群中的基本单元，拥有唯一标识符和通信地址。
  2. 成员管理 (Membership) - 维护集群成员信息和状态。
  3. Gossip 协议 - 实现高效的信息传播，保证成员信息的最终一致性。
  4. 传输层 (Transport) - 负责节点间通信，默认基于 HTTP。
  5. 事件系统 - 处理节点加入、离开等集群事件。

集群形成过程​

1. 初始化节点​

创建节点时，配置基本信息并初始化核心组件：

// 创建新的集群节点
node, err := cluster.NewNode(
    "node-1",                      // 节点 ID
    localCache,                    // 本地缓存实例
    cluster.WithBindAddr(":7946"), // 绑定地址
    cluster.WithGossipInterval(500*time.Millisecond), // Gossip 间隔
    cluster.WithProbeInterval(1*time.Second),         // 探测间隔
)

初始化过程包括：

  • 创建成员管理器
  • 初始化 Gossip 协议
  • 设置传输层
  • 创建事件发布器

2. 节点启动​

调用 Start() 方法启动节点服务：

err := node.Start()
if err != nil {
    log.Fatalf("Failed to start node: %v", err)
}

此过程会依次启动：

  1. 传输层服务，开始监听通信端口
  2. 成员管理服务，初始化本地成员信息
  3. Gossip 协议，准备传播成员信息
  4. 同步管理器，准备数据同步

3. 集群引导与加入​

FyerCache 支持两种集群形成方式：

自举集群（第一个节点）​

第一个节点通过自身 IP 地址加入自己创建的集群：

err := node.Join(node.config.BindAddr)

此时节点只看到自己作为集群成员。

加入现有集群​

后续节点通过提供任一现有节点的地址来加入集群：

err := node.Join("192.168.1.100:7946") // 种子节点地址

加入集群的过程包括：

  1. 创建加入消息并发送到种子节点
  2. 种子节点返回其所知的集群成员信息
  3. 将种子节点添加到本地成员列表
  4. 通过 Gossip 协议开始与其他节点交换信息
  5. 触发与种子节点的数据同步 (如需要)

4. Gossip 传播​

成员信息通过 Gossip 协议传播到整个集群：

// Gossip 周期性传播过程
func (g *Gossiper) spreadGossip() error {
    // 获取当前成员
    members := g.membership.Members()
    
    // 为所有已知节点准备 Gossip 负载
    payload := createGossipPayload(members)
    
    // 选择目标节点并发送消息
    targetNodes := g.selectRandomNodes(g.fanout, members)
    for _, target := range targetNodes {
        g.transport.Send(target.Addr, createGossipMessage(payload))
    }
    
    return nil
}

Gossip 协议确保：

  • 新节点信息最终传播到所有节点
  • 节点状态变化同步到整个集群
  • 系统具有良好的可扩展性，通信复杂度保持稳定

成员管理与发现​

节点状态监控​

FyerCache 跟踪集群中各节点的状态，支持以下状态：

const (
    NodeStatusUp NodeStatus = iota      // 正常运行
    NodeStatusSuspect                   // 可疑状态（可能失效）
    NodeStatusDown                      // 已失效
    NodeStatusLeft                      // 已主动离开
)

健康检查机制​

FyerCache 使用主动探测和 Gossip 结合的方式检测节点健康状态：

// 节点探测逻辑
func (m *Membership) probeNextNode() {
    // 选择下一个要探测的节点
    nodeInfo := m.selectNextProbeTarget()
    
    // 创建 Ping 消息
    pingMsg := createPingMessage()
    
    // 发送 Ping 并等待响应
    resp, err := m.transport.Send(nodeInfo.Addr, pingMsg)
    
    // 处理响应或超时
    if err != nil {
        // 标记为可疑状态
        m.MarkSuspect(nodeInfo.ID)
    } else {
        // 标记为活跃状态
        m.MarkAlive(nodeInfo.ID)
    }
}

健康检查流程：

  1. 定期向集群中的其他节点发送 Ping 消息
  2. 如果节点响应，标记为"活跃"状态 (UP)
  3. 如果节点未能响应，标记为"可疑"状态 (SUSPECT)
  4. 如果节点在一定时间内持续未响应，标记为"失效"状态 (DOWN)

可疑状态机制​

可疑状态是一种中间状态，避免因网络短暂问题误判节点失效：

func (m *Membership) MarkSuspect(nodeID string) {
    m.mu.Lock()
    defer m.mu.Unlock()
    
    if info, exists := m.members[nodeID]; exists {
        info.Status = cache.NodeStatusSuspect
        m.suspectTime[nodeID] = time.Now()
        // 发布状态变更事件
    }
}

// 探测循环定期检查可疑节点
func (m *Membership) probeLoop() {
    for {
        // ...
        // 检查可疑节点是否超过阈值时间
        for nodeID, suspectTime := range m.suspectTime {
            if time.Since(suspectTime) > m.suspicionMult*m.probeInterval {
                m.MarkFailed(nodeID, "probe timeout exceeded")
            }
        }
        // ...
    }
}

可疑状态节点：

  1. 继续接收探测，有机会恢复到"活跃"状态
  2. 如果超过 suspicionMult * probeInterval 时间未能恢复，被标记为"失效"
  3. 失效状态通过 Gossip 传播到其他节点，解决"拜占庭将军"问题

节点发现机制​

基于 Gossip 的发现​

新节点加入集群后，通过以下机制被其他节点发现：

  1. 直接发现 - 与种子节点通信时直接被种子节点发现
  2. 间接发现 - 通过 Gossip 协议传播成员信息
  3. 周期性更新 - 定期 Gossip 交换确保最终一致性

Gossip 消息包含节点 ID、地址、状态和其他元数据：

// GossipNodeInfo Gossip传播的节点信息
type GossipNodeInfo struct {
    ID       string     `json:"id"`
    Addr     string     `json:"addr"`
    Status   NodeStatus `json:"status"`
    LastSeen time.Time  `json:"last_seen"`
    Metadata map[string]string `json:"metadata,omitempty"`
}

节点离开处理​

节点可以通过两种方式离开集群：

  1. 优雅离开 - 节点主动通知其离开意图：
// 优雅离开集群
err := node.Leave()
  1. 失败检测 - 节点未能回应探测，被标记为失效：
// 失败检测通过成员管理器的探测机制完成
m.MarkFailed(nodeID, "probe timeout exceeded")

当节点离开或失效时：

  • 其状态被更新并通过 Gossip 传播
  • 集群更新其内部视图
  • 数据重新分布到剩余节点

配置选项​

FyerCache 提供多种配置选项，用于调整集群行为：

// 创建带自定义配置的集群节点
node, err := cluster.NewNode(
    "node-1",
    localCache,
    // 节点标识和通信
    cluster.WithNodeID("node-1"),
    cluster.WithBindAddr("192.168.1.101:7946"),
    
    // Gossip 协议配置
    cluster.WithGossipInterval(500*time.Millisecond),
    
    // 健康检查配置
    cluster.WithProbeInterval(1*time.Second),
    cluster.WithProbeTimeout(500*time.Millisecond),
    cluster.WithSuspicionMult(3),
    
    // 事件系统配置
    cluster.WithEventBufferSize(100),
    
    // 重试策略
    cluster.WithMaxRetries(3),
    cluster.WithRetryInterval(200*time.Millisecond),
)

关键配置参数​

参数描述默认值建议值
BindAddr节点绑定地址，格式为 IP:端口:7946根据网络环境配置
GossipIntervalGossip 消息交换间隔1s0.5s - 2s
ProbeInterval健康检查间隔1s1s - 5s
ProbeTimeout探测超时时间500ms100ms - 1s
SuspicionMult可疑状态乘数33 - 5
EventBufferSize事件缓冲区大小10050 - 500

事件系统​

FyerCache 实现了事件系统，允许应用程序响应集群变化：

// 订阅集群事件
eventChannel := node.Events()
go func() {
    for event := range eventChannel {
        switch event.Type {
        case cache.EventNodeJoin:
            log.Printf("Node joined: %s", event.NodeID)
        case cache.EventNodeLeave:
            log.Printf("Node left: %s", event.NodeID)
        case cache.EventNodeFailed:
            log.Printf("Node failed: %s, reason: %v", event.NodeID, event.Details)
        case cache.EventNodeRecovered:
            log.Printf("Node recovered: %s", event.NodeID)
        }
    }
}()

支持的事件类型：

const (
    EventNodeJoin NodeEventType = iota     // 节点加入
    EventNodeLeave                         // 节点离开
    EventNodeFailed                        // 节点失效
    EventNodeRecovered                     // 节点恢复
    EventLeaderElected                     // 领导者选举（在主从模式中）
    EventSyncCompleted                     // 同步完成
)

示例：集群形成过程​

以下代码演示三节点集群的形成过程：

package main

import (
    "log"
    "time"
    
    "github.com/fyerfyer/fyer-cache/cache"
    "github.com/fyerfyer/fyer-cache/cache/cluster"
)

func main() {
    // 创建三个缓存节点
    node1, err := createNode("node1", ":7946")
    if err != nil {
        log.Fatalf("Failed to create node1: %v", err)
    }
    defer node1.Stop()
    
    node2, err := createNode("node2", ":7947")
    if err != nil {
        log.Fatalf("Failed to create node2: %v", err)
    }
    defer node2.Stop()
    
    node3, err := createNode("node3", ":7948")
    if err != nil {
        log.Fatalf("Failed to create node3: %v", err)
    }
    defer node3.Stop()
    
    // 1. 启动所有节点
    if err := node1.Start(); err != nil {
        log.Fatalf("Failed to start node1: %v", err)
    }
    if err := node2.Start(); err != nil {
        log.Fatalf("Failed to start node2: %v", err)
    }
    if err := node3.Start(); err != nil {
        log.Fatalf("Failed to start node3: %v", err)
    }
    
    // 2. 第一个节点形成自己的集群
    if err := node1.Join(node1.config.BindAddr); err != nil {
        log.Fatalf("Failed to bootstrap first node: %v", err)
    }
    
    log.Println("Node1 formed its own cluster")
    
    // 3. 其他节点加入第一个节点的集群
    if err := node2.Join("127.0.0.1:7946"); err != nil {
        log.Fatalf("Node2 failed to join: %v", err)
    }
    log.Println("Node2 joined the cluster")
    
    // 给 Gossip 一些时间传播
    time.Sleep(1 * time.Second)
    
    if err := node3.Join("127.0.0.1:7946"); err != nil {
        log.Fatalf("Node3 failed to join: %v", err)
    }
    log.Println("Node3 joined the cluster")
    
    // 4. 等待集群稳定
    time.Sleep(2 * time.Second)
    
    // 5. 检查每个节点的成员信息
    for i, node := range []*cluster.Node{node1, node2, node3} {
        members := node.Members()
        log.Printf("Node%d sees %d members", i+1, len(members))
        for _, m := range members {
            log.Printf(" - Member: %s, Status: %v", m.ID, m.Status)
        }
    }
}

func createNode(id, addr string) (*cluster.Node, error) {
    // 创建本地缓存实例
    localCache := cache.NewMemoryCache()
    
    // 创建集群节点
    node, err := cluster.NewNode(
        id, 
        localCache,
        cluster.WithBindAddr(addr),
        cluster.WithGossipInterval(500*time.Millisecond),
        cluster.WithProbeInterval(1*time.Second),
        cluster.WithSuspicionMult(3),
    )
    
    return node, err
}
