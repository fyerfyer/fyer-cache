Membership Management

成员管理是 FyerCache 集群功能的核心组件，负责维护集群成员信息、监测节点健康状态并处理节点的加入和离开。

核心组件​

FyerCache 的成员管理由三个核心组件组成，它们协同工作确保集群的稳定运行：

┌───────────────────────────────────┐
│           Membership              │
│   (成员列表维护和状态管理)          │
└─────────────────┬─────────────────┘
                  │
                  ▼
┌───────────────────────────────────┐
│            Gossiper               │
│      (成员信息的传播与交换)         │
└─────────────────┬─────────────────┘
                  │
                  ▼
┌───────────────────────────────────┐
│         EventPublisher            │
│      (成员事件通知与分发)          │
└───────────────────────────────────┘

1. Membership​

Membership 是成员管理的核心组件，维护集群中所有节点的信息和状态：

// Membership 集群成员管理
type Membership struct {
    localID string            // 本地节点ID
    localAddr string          // 本地节点地址
    members map[string]*cache.NodeInfo // 所有节点信息
    suspectTime map[string]time.Time  // 可疑状态时间记录
    suspicionMult int        // 可疑状态超时倍数
    probeInterval time.Duration  // 健康检查间隔
    probeTimeout time.Duration   // 健康检查超时
    probeIndex int            // 当前探测索引
    events *EventPublisher    // 事件发布器
    transport NodeTransporter // 传输层
    mu sync.RWMutex          // 互斥锁
    stopCh chan struct{}     // 停止信号
    running bool             // 运行状态
}

2. Gossiper​

Gossiper 实现 Gossip 协议，负责成员信息的高效传播：

// Gossiper 实现Gossip协议
type Gossiper struct {
    localID string           // 本地节点ID
    membership *Membership   // 成员管理器
    transport NodeTransporter // 传输层
    interval time.Duration   // gossip间隔
    fanout int               // 每轮选择的节点数
    maxPacketSize int        // 最大消息大小
    rand *rand.Rand          // 随机数生成器
    randMu sync.Mutex        // 随机数锁
    stopCh chan struct{}     // 停止信号
    running bool             // 运行状态
    mu sync.Mutex            // 互斥锁
}

3. EventPublisher​

EventPublisher 提供事件通知系统，发布集群成员变化事件：

// EventPublisher 事件发布器
type EventPublisher struct {
    eventCh chan cache.ClusterEvent        // 事件通道
    subscribers []chan<- cache.ClusterEvent // 订阅者列表
    mu sync.RWMutex                       // 互斥锁
    bufferSize int                         // 缓冲区大小
}

节点状态管理​

FyerCache 定义了四种节点状态，用于跟踪集群成员的生命周期：

const (
    NodeStatusUp NodeStatus = iota      // 正常运行
    NodeStatusSuspect                   // 可疑状态（可能失效）
    NodeStatusDown                      // 已失效
    NodeStatusLeft                      // 已主动离开
)

状态转换流程​

节点状态的转换遵循以下流程：

  1. Up → Suspect：当节点无法响应健康检查时
  2. Suspect → Down：当节点在可疑状态超过阈值时间后仍无响应
  3. Suspect → Up：当可疑状态节点恢复响应时
  4. Up → Left：当节点主动离开集群时

可疑状态机制​

"可疑"状态是一种中间状态，避免因网络暂时性问题误判节点失效：

// MarkSuspect 将节点标记为可疑状态
func (m *Membership) MarkSuspect(nodeID string) {
    m.mu.Lock()
    defer m.mu.Unlock()
    
    if info, exists := m.members[nodeID]; exists {
        info.Status = cache.NodeStatusSuspect
        m.suspectTime[nodeID] = time.Now()
        // 发布状态变更事件
        m.events.Publish(NewNodeSuspectEvent(nodeID))
    }
}

只有当节点在"可疑"状态持续一段时间仍无响应时，才会被标记为"失效"：

// 在探测循环中检查可疑节点
for nodeID, suspectTime := range m.suspectTime {
    if time.Since(suspectTime) > m.suspicionMult*m.probeInterval {
        m.MarkFailed(nodeID, "probe timeout exceeded")
    }
}

健康检查机制​

FyerCache 使用主动探测机制检测节点健康状态：

定期探测​

// probeLoop 定期进行节点健康检查
func (m *Membership) probeLoop() {
    ticker := time.NewTicker(m.probeInterval)
    defer ticker.Stop()

    for {
        select {
        case <-m.stopCh:
            return
        case <-ticker.C:
            m.probeNextNode()
        }
    }
}

节点探测​

探测使用 Ping/Pong 机制检测节点响应能力：

// probeNextNode 探测下一个节点是否健康
func (m *Membership) probeNextNode() {
    m.mu.RLock()
    members := m.Members()
    m.mu.RUnlock()
    
    // 跳过本地节点和非活跃节点
    // ...

    // 创建 Ping 消息
    pingPayload := &PingPayload{
        Sequence: time.Now().UnixNano(),
    }
    
    // 发送 Ping 并等待响应
    resp, err := m.transport.Send(nodeInfo.Addr, pingMsg)
    
    // 处理响应或超时
    if err != nil {
        // 标记为可疑状态
        m.MarkSuspect(nodeInfo.ID)
    } else {
        // 标记为活跃状态
        m.MarkAlive(nodeInfo.ID)
    }
}

超时配置​

可以通过选项配置探测间隔和超时时间：

node, err := cluster.NewNode(
    "node-1", 
    cache,
    cluster.WithProbeInterval(1*time.Second),   // 探测间隔
    cluster.WithProbeTimeout(500*time.Millisecond), // 探测超时
    cluster.WithSuspicionMult(3),               // 可疑状态乘数
)

Gossip 信息传播​

FyerCache 使用 Gossip 协议在集群中传播成员信息：

工作原理​

  1. 每个节点定期选择几个随机节点
  2. 向选中的节点发送自己所知道的成员信息
  3. 接收节点更新自己的成员视图
  4. 信息以指数速度在集群中传播
  5. 所有节点最终达到一致状态

实现细节​

// spreadGossip 传播成员信息给随机节点
func (g *Gossiper) spreadGossip() error {
    // 获取当前成员
    members := g.membership.Members()
    
    var otherNodes []cache.NodeInfo
    for _, member := range members {
        if member.ID != g.localID && member.Status != cache.NodeStatusDown && member.Status != cache.NodeStatusLeft {
            otherNodes = append(otherNodes, member)
        }
    }
    
    // 如果没有其他节点，跳过本轮传播
    if len(otherNodes) == 0 {
        return nil
    }
    
    // 准备 Gossip 负载
    payload := createGossipPayload(members)
    
    // 选择目标节点
    targets := g.selectRandomNodes(g.fanout, otherNodes)
    
    // 发送 Gossip 消息给选中的节点
    for _, target := range targets {
        go func(addr string) {
            // 异步发送以避免阻塞
            g.transport.Send(addr, gossipMsg)
        }(target.Addr)
    }
    
    return nil
}

Gossip 配置​

可以配置 Gossip 间隔和扇出数量（每次选择的节点数）：

node, err := cluster.NewNode(
    "node-1", 
    cache,
    cluster.WithGossipInterval(500*time.Millisecond), // Gossip 间隔
)

节点加入与离开​

节点加入过程​

  1. 新节点连接一个已知节点（种子节点）
  2. 新节点发送加入请求
  3. 种子节点返回当前集群成员信息
  4. 新节点被添加到集群成员列表中
  5. 节点加入的信息通过 Gossip 协议传播到整个集群
// Join 加入集群
func (n *Node) Join(seedNodeAddr string) error {
    // 创建加入消息
    payload := &JoinPayload{
        NodeID:   n.id,
        Address:  n.transport.Address(),
    }
    
    // 序列化并发送消息
    joinMsg := &cache.NodeMessage{
        Type:     cache.MsgJoin,
        SenderID: n.id,
        Timestamp: time.Now(),
        Payload: payloadBytes,
    }
    
    // 发送加入请求
    resp, err := n.transport.Send(seedNodeAddr, joinMsg)
    
    // 处理响应，添加种子节点
    n.membership.AddMember(respPayload.NodeID, seedNodeAddr, nil)
    
    return nil
}

节点离开过程​

  1. 节点发送离开消息给集群中的其他节点
  2. 节点被标记为"已离开"状态
  3. 节点离开的信息通过 Gossip 协议传播
  4. 其他节点将该节点从活跃成员列表中移除
// Leave 离开集群
func (n *Node) Leave() error {
    // 创建离开消息
    payload := &LeavePayload{
        NodeID:   n.id,
        Graceful: true,
    }
    
    // 序列化并发送消息到所有成员
    members := n.membership.Members()
    for _, member := range members {
        if member.ID != n.id {
            n.transport.Send(member.Addr, leaveMsg)
        }
    }
    
    // 标记自己为离开状态
    n.membership.RemoveMember(n.id, true)
    
    return nil
}

事件系统​

FyerCache 提供事件通知系统，使应用程序能够响应成员状态变化：

事件类型​

const (
    EventNodeJoin NodeEventType = iota     // 节点加入
    EventNodeLeave                         // 节点离开
    EventNodeFailed                        // 节点失效
    EventNodeRecovered                     // 节点恢复
    EventLeaderElected                     // 领导者选举
    EventSyncCompleted                     // 同步完成
)

订阅事件​

// 订阅集群事件
eventChannel := node.Events()
go func() {
    for event := range eventChannel {
        switch event.Type {
        case cache.EventNodeJoin:
            log.Printf("Node joined: %s", event.NodeID)
        case cache.EventNodeLeave:
            log.Printf("Node left: %s", event.NodeID)
        case cache.EventNodeFailed:
            log.Printf("Node failed: %s", event.NodeID)
        case cache.EventNodeRecovered:
            log.Printf("Node recovered: %s", event.NodeID)
        }
    }
}()

事件发布​

当节点状态变化时，系统会发布相应事件：

// 发布节点加入事件
m.events.Publish(NewNodeJoinEvent(nodeID, addr))

// 发布节点失效事件
m.events.Publish(NewNodeFailedEvent(nodeID, reason))

协调者机制​

FyerCache 实现了简单的协调者选择机制，用于特定任务的协调：

// IsCoordinator 检查当前节点是否为协调者
// 采用简单策略：节点ID字母顺序最小的为协调者
func (m *Membership) IsCoordinator() bool {
    m.mu.RLock()
    defer m.mu.RUnlock()
    
    localID := m.localID
    
    for id := range m.members {
        // 如果有ID比本地ID小的节点，则本地节点不是协调者
        if id < localID {
            return false
        }
    }
    
    // 如果本地ID是最小的，则是协调者
    return true
}

这种机制保证在任何时刻，集群中只有一个节点被认为是协调者，用于处理需要全局协调的任务。

使用示例​

创建和启动集群节点​

// 创建节点
node, err := cluster.NewNode(
    "node-1",                      // 节点 ID
    localCache,                    // 本地缓存实例
    cluster.WithBindAddr(":7946"), // 绑定地址
    cluster.WithGossipInterval(500*time.Millisecond), // Gossip 间隔
    cluster.WithProbeInterval(1*time.Second),         // 探测间隔
)
if err != nil {
    log.Fatalf("Failed to create node: %v", err)
}

// 启动节点
if err := node.Start(); err != nil {
    log.Fatalf("Failed to start node: %v", err)
}

// 确保节点优雅停止
defer node.Stop()

引导新集群​

// 第一个节点自举形成集群
if err := node.Join(node.transport.Address()); err != nil {
    log.Fatalf("Failed to bootstrap cluster: %v", err)
}

加入现有集群​

// 加入已存在的集群
if err := node.Join("192.168.1.100:7946"); err != nil {
    log.Fatalf("Failed to join cluster: %v", err)
}

监控集群成员​

// 获取当前集群成员
members := node.Members()
fmt.Printf("Cluster has %d members\n", len(members))

for _, member := range members {
    fmt.Printf("Member: %s, Address: %s, Status: %s\n", 
               member.ID, member.Addr, getStatusString(member.Status))
}

优雅离开集群​

// 离开集群
if err := node.Leave(); err != nil {
    log.Printf("Error leaving cluster: %v", err)
}
