Memcache Implementation

内存缓存是 FyerCache 的基础构建块，提供高性能的内存中键值存储，支持过期时间和自动清理。

核心设计​

FyerCache 的内存缓存实现基于以下设计原则：

  1. 高并发性能 - 通过分片减少锁竞争，支持高并发读写
  2. 低延迟 - 优化关键路径，确保读写操作的最小开销
  3. 灵活的过期机制 - 支持 TTL 和自动清理过期项
  4. 内存效率 - 可选的内存限制和淘汰策略
  5. 扩展性 - 支持自定义淘汰回调和性能监控

MemoryCache - 基本内存缓存​

MemoryCache 是 FyerCache 的基础缓存实现，提供标准的 Get、Set 和 Del 操作。它使用分片存储提高并发性能，并支持 TTL 过期和自动清理机制。

核心数据结构​

// MemoryCache 内存缓存结构
type MemoryCache struct {
    data        *ShardedMap                 // 存储缓存数据，使用分片映射代替sync.Map
    onEvict     func(key string, value any) // 当缓存项被移除时的回调函数
    stopCleanUp chan struct{}               // 停止清理任务
    cleaner     *Cleaner                    // 异步清理器
    
    // 配置字段
    shardCount      int           // 分片数量
    useAsyncCleanup bool          // 是否使用异步清理
    workerCount     int           // 工作协程数量
    queueSize       int           // 任务队列大小
    cleanupInterval time.Duration // 清理间隔
    stats           *StatsCollector // 统计收集器
}

// cacheItem 缓存项结构
type cacheItem struct {
    value      any       // 实际存储的值
    expiration time.Time // 过期时间
}

缓存操作​

Set 操作​

Set 方法将键值对存储到缓存中，可设置过期时间：

func (c *MemoryCache) Set(ctx context.Context, key string, value any, expiration time.Duration) error {
    // 构造缓存项
    item := &cacheItem{
        value: value,
    }
    
    // 设置过期时间（如果提供）
    if expiration > 0 {
        item.expiration = time.Now().Add(expiration)
    }
    
    // 存储到分片映射中
    c.data.Store(key, item)
    
    return nil
}

Get 操作​

Get 方法从缓存检索值，处理过期检查：

func (c *MemoryCache) Get(ctx context.Context, key string) (any, error) {
    // 从分片映射加载
    item, found := c.data.Load(key)
    if !found {
        return nil, ferr.ErrKeyNotFound
    }
    
    // 检查是否过期
    if !item.expiration.IsZero() && item.expiration.Before(time.Now()) {
        // 异步删除过期项
        go c.Del(ctx, key)
        return nil, ferr.ErrKeyNotFound
    }
    
    return item.value, nil
}

Del 操作​

Del 方法从缓存中删除键值对：

func (c *MemoryCache) Del(ctx context.Context, key string) error {
    c.data.Delete(key)
    return nil
}

分片设计​

为了最大化并发性能，FyerCache 使用 ShardedMap 分片映射替代标准的 sync.Map。分片设计可以显著减少锁竞争，特别是在高并发环境中。

ShardedMap 实现​

// ShardedMap 分片映射结构
type ShardedMap struct {
    shards    []*Shard
    count     int    // 分片数量
    mask      uint32 // 用于位运算快速定位分片
    itemCount int64  // 总项目数量
}

// Shard 单个分片
type Shard struct {
    items map[string]*cacheItem
    mu    sync.RWMutex // 每个分片有独立的读写锁
}

分片算法​

FyerCache 使用快速哈希算法将键分配到适当的分片：

// getShard 根据键获取对应的分片
func (sm *ShardedMap) getShard(key string) *Shard {
    hasher := fnv.New32()
    hasher.Write([]byte(key))
    hash := hasher.Sum32()

    // 使用位掩码快速计算分片索引
    index := hash & sm.mask
    return sm.shards[index]
}

这种设计确保了：

  1. 键的分布均匀性，避免热点
  2. 每个分片有独立的锁，减少全局锁竞争
  3. 位掩码操作的高效性（比取模运算更快）

清理机制​

FyerCache 支持两种清理机制来移除过期项：同步清理和异步清理。

同步清理​

同步清理在单一协程中定期执行，适用于简单场景：

func (c *MemoryCache) startSyncCleanupTimer(interval time.Duration) {
    c.stopCleanUp = make(chan struct{})
    ticker := time.NewTicker(interval)
    
    go func() {
        for {
            select {
            case <-ticker.C:
                c.cleanUp()
            case <-c.stopCleanUp:
                ticker.Stop()
                return
            }
        }
    }()
}

异步清理​

对于大规模缓存，FyerCache 提供异步清理机制，使用工作池并行处理清理任务：

// Cleaner 清理器结构
type Cleaner struct {
    cache       *ShardedMap       // 要清理的分片缓存
    taskQueue   chan CleanTask    // 任务队列
    workerCount int               // 工作协程数量
    interval    time.Duration     // 清理周期
    stopChan    chan struct{}     // 停止信号
    wg          sync.WaitGroup    // 用于等待所有worker完成
    onEvict     func(string, any) // 淘汰回调
    running     bool              // 是否已启动
    mu          sync.Mutex        // 保护running状态
}

异步清理的工作流程：

  1. 每个清理周期，为每个分片创建清理任务
  2. 多个工作协程从任务队列消费，并行处理清理
  3. 每个工作协程处理特定分片的过期项删除

这种设计避免了单一清理协程阻塞缓存操作，同时通过并行处理提高了清理效率。

LRU 淘汰策略​

当内存使用超过限制时，FyerCache 使用 LRU 算法淘汰最久未使用的项：

// Evict LRU算法实现
func (lru *LRUEviction) Evict(c *MemCacheMemory) string {
    lru.mu.RLock()
    defer lru.mu.RUnlock()
    
    // 选择最久未使用的节点
    if lru.accessList.tail.prev != lru.accessList.head {
        victim := lru.accessList.tail.prev
        return victim.key
    }
    
    return ""
}

LRU 访问列表实现：

  • 使用双向链表跟踪访问顺序
  • 新访问的项移到列表头部
  • 最久未使用的项位于列表尾部
  • 当需要淘汰时，从尾部删除

性能优化​

FyerCache 采用多种优化技术提高性能：

1. 分片锁​

分片锁是最关键的优化，将数据分散到多个分片，每个分片有独立的锁：

  • 分片数量默认为 32，通常设置为 CPU 核心数的倍数
  • 分片数量必须是 2 的幂，便于使用位掩码快速定位分片

2. 读写分离​

使用 sync.RWMutex 实现读写分离：

  • 读操作可以并发执行
  • 写操作需要独占锁

3. 异步清理​

将清理过程与正常缓存操作分离：

  • 避免清理阻塞缓存访问
  • 使用工作池提高清理吞吐量

4. 位操作优化​

使用位操作代替取模运算：

// 使用位掩码快速计算分片索引 
index := hash & sm.mask  // 比 hash % shardCount 更高效

5. 延迟删除​

Get 方法中检测到过期项时不阻塞返回：

if !item.expiration.IsZero() && item.expiration.Before(time.Now()) {
    // 异步删除过期项
    go c.Del(ctx, key)
    return nil, ferr.ErrKeyNotFound
}

统计与监控​

MemoryCache 内置了基本的统计功能：

// 获取缓存项目数量
func (c *MemoryCache) ItemCount() int64 {
    return c.data.Count()
}

// 获取缓存命中率
func (c *MemoryCache) HitRate() float64 {
    return c.stats.HitRate()
}

// 获取缓存未命中率
func (c *MemoryCache) MissRate() float64 {
    return c.stats.MissRate()
}

这些方法可以与监控系统集成，通过 FyerCache 的监控模块提供实时性能指标。

使用示例​

基本内存缓存​

// 创建默认配置的内存缓存
cache := cache.NewMemoryCache()

ctx := context.Background()

// 设置带过期时间的缓存项
cache.Set(ctx, "session:123", sessionData, 30*time.Minute)

// 获取缓存值
value, err := cache.Get(ctx, "session:123")

// 删除缓存项
cache.Del(ctx, "session:123")

自定义配置​

// 创建自定义配置的内存缓存
cache := cache.NewMemoryCache(
    // 设置64个分片减少锁竞争
    cache.WithShardCount(64),
    
    // 配置淘汰回调函数
    cache.WithEvictionCallback(func(key string, value any) {
        log.Printf("Evicted: %s", key)
    }),
    
    // 启用异步清理
    cache.WithAsyncCleanup(true),
    
    // 设置清理间隔
    cache.WithCleanupInterval(10*time.Minute),
    
    // 配置清理工作协程数
    cache.CacheWithWorkerCount(runtime.NumCPU()),
)

// 记得在应用退出前关闭缓存资源
defer cache.Close()

带内存限制的缓存​

// 创建基础内存缓存
baseCache := cache.NewMemoryCache()

// 创建最大使用200MB内存的缓存装饰器
memCache := cache.NewMemCacheMemory(baseCache, 200*1024*1024)

// memCache 会自动使用LRU策略淘汰项目当内存使用超过限制
