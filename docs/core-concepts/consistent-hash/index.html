Consistent Hashing Principles

一致性哈希是分布式缓存系统中一种关键算法，它解决了传统哈希算法在节点变化时需要大量数据迁移的问题。

基本原理​

传统哈希的局限性​

传统的数据分片方式通常使用取模运算：hash(key) % N（N是节点数量）。这种方式在节点数量发生变化时，会导致大量键需要重新映射：

当N从3变为4时:
hash(key) % 3 = 0 的键 -> 现在可能映射到0、1、2或3
hash(key) % 3 = 1 的键 -> 现在可能映射到0、1、2或3
hash(key) % 3 = 2 的键 -> 现在可能映射到0、1、2或3

在最坏情况下，几乎所有键都需要重新分配，导致缓存大规模失效。

一致性哈希的解决方案​

一致性哈希算法通过将节点和键映射到一个固定的环上，使得节点变化时只影响紧邻节点的数据分布。FyerCache的一致性哈希实现遵循以下步骤：

  1. 将0到2^32-1的整数空间映射到一个环上
  2. 使用哈希函数将节点ID映射到环上的点
  3. 将每个键通过同样的哈希函数映射到环上
  4. 每个键分配给其顺时针方向遇到的第一个节点

当节点加入或离开时，只有该节点与前一个节点之间的键需要重新映射，大大减少了数据迁移。

FyerCache中的实现​

FyerCache中一致性哈希的核心结构是ConsistentHash：

// ConsistentHash 一致性哈希环实现
type ConsistentHash struct {
    // 哈希函数
    hashFunc HashFunc

    // 每个物理节点对应的虚拟节点数量
    replicas int

    // 已排序的哈希值
    sortedHashes []uint32

    // 哈希值到节点的映射
    hashMap map[uint32]string

    // 节点权重
    weights map[string]int

    // 节点到其哈希值的映射，用于高效删除
    nodeHashes map[string][]uint32

    // 保护环的并发访问
    mu sync.RWMutex
}

虚拟节点​

FyerCache实现了虚拟节点机制，每个物理节点在环上对应多个点，显著提高了数据分布的均匀性：

// 为节点创建虚拟节点
for i := 0; i < c.replicas*weight; i++ {
    // 为虚拟节点创建唯一标识
    virtualNodeName := fmt.Sprintf("%s:%d", node, i)
    
    // 计算哈希值
    hash := c.hashFunc([]byte(virtualNodeName))
    
    // 添加到排序列表和映射
    c.sortedHashes = append(c.sortedHashes, hash)
    c.hashMap[hash] = node
    nodeHashes = append(nodeHashes, hash)
}

默认情况下，每个物理节点会创建100个虚拟节点，这个值可以通过配置选项进行调整。

节点权重​

FyerCache支持为节点指定不同的权重，权重越高的节点会在环上占据更多位置，承担更多数据：

// Add 添加节点到哈希环
func (c *ConsistentHash) Add(node string, weight int) {
    // ...
    // 记录节点权重
    c.weights[node] = weight
    
    // 创建虚拟节点数量与权重成正比
    for i := 0; i < c.replicas*weight; i++ {
        // ...
    }
}

节点查找​

键的路由逻辑核心在Get方法中，它使用二分查找快速定位键应该存储在哪个节点：

// Get 根据键获取对应的节点
func (c *ConsistentHash) Get(key string) string {
    // 计算键的哈希值
    hash := c.hashFunc([]byte(key))
    
    // 二分查找大于等于该哈希值的第一个索引
    idx := c.search(hash)
    
    // 返回对应的节点
    return c.hashMap[c.sortedHashes[idx]]
}

如果所有节点的哈希值都小于键的哈希值，则返回第一个节点，形成环形结构。

数据复制机制​

为提高系统可用性，FyerCache支持将数据复制到多个节点。GetN方法返回键顺时针方向遇到的N个唯一节点：

// GetN 获取键映射到的N个节点，用于备份
func (c *ConsistentHash) GetN(key string, count int) []string {
    if count <= 0 {
        return nil
    }
    
    c.mu.RLock()
    defer c.mu.RUnlock()
    
    if len(c.sortedHashes) == 0 {
        return nil
    }
    
    // 计算键的哈希值
    hash := c.hashFunc([]byte(key))
    
    // 二分查找起始位置
    idx := c.search(hash)
    
    // 收集不重复的节点
    uniqueNodes := make(map[string]bool)
    result := make([]string, 0, count)
    
    for len(uniqueNodes) < count && len(uniqueNodes) < len(c.weights) {
        node := c.hashMap[c.sortedHashes[idx]]
        if !uniqueNodes[node] {
            uniqueNodes[node] = true
            result = append(result, node)
        }
        idx = (idx + 1) % len(c.sortedHashes)
    }
    
    return result
}

节点权重示例​

FyerCache允许为节点设置不同权重：

// 创建节点分片缓存
shardedCache := cache.NewNodeShardedCache()

// 添加三个节点，权重不同
shardedCache.AddNode("node1", cache1, 1)  // 权重1
shardedCache.AddNode("node2", cache2, 2)  // 权重2
shardedCache.AddNode("node3", cache3, 3)  // 权重3

上例中，理想情况下键的分布比例应接近1:2:3。实际测试中，使用100个虚拟节点时，键分布比例约为16.5%:32.8%:50.7%，非常接近理想值。

并发安全性​

FyerCache的一致性哈希实现是并发安全的，通过读写锁保护关键操作：

func (c *ConsistentHash) Add(node string, weight int) {
    c.mu.Lock()
    defer c.mu.Unlock()
    // ...
}

func (c *ConsistentHash) Get(key string) string {
    c.mu.RLock()
    defer c.mu.RUnlock()
    // ...
}

在高并发场景中，读操作使用读锁，不会相互阻塞；写操作（添加/删除节点）相对较少，对性能影响有限。

二分查找优化​

FyerCache使用二分查找算法快速定位键的位置，这在环上有大量虚拟节点时尤为重要：

// search 二分查找大于等于给定哈希值的第一个索引
func (c *ConsistentHash) search(hash uint32) int {
    // 二分查找
    i, j := 0, len(c.sortedHashes)
    for i < j {
        mid := (i + j) / 2
        if c.sortedHashes[mid] < hash {
            i = mid + 1
        } else {
            j = mid
        }
    }
    
    // 如果没有找到大于等于哈希的值，回到环的起点
    if i >= len(c.sortedHashes) {
        i = 0
    }
    
    return i
}

实际应用示例​

在FyerCache中使用一致性哈希分片的典型代码如下：

// 创建基于节点的分片缓存
shardedCache := cache.NewNodeShardedCache(
    cache.WithNodeHashReplicas(200),     // 设置虚拟节点数量
    cache.WithNodeReplicaFactor(2),      // 设置副本因子(数据冗余)
)

// 添加3个缓存节点
node1 := cache.NewMemoryCache()
node2 := cache.NewMemoryCache()
node3 := cache.NewMemoryCache()

shardedCache.AddNode("node1", node1, 1)
shardedCache.AddNode("node2", node2, 2)
shardedCache.AddNode("node3", node3, 1)

// 写入数据，自动路由到正确节点
shardedCache.Set(ctx, "user:1001", userData, time.Hour)

// 读取数据，自动从正确节点获取
userData, err := shardedCache.Get(ctx, "user:1001")
