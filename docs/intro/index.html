fyer-cache

FyerCache 是一个高性能、灵活且可扩展的 Go 语言缓存系统，提供从单机内存缓存到复杂分布式缓存集群的完整解决方案。

设计理念​

FyerCache 的设计基于以下关键理念：

  • 简单性与灵活性 - 提供简洁直观的接口，同时通过选项模式支持高度自定义
  • 分层架构 - 从内存缓存到分布式集群的渐进式功能层次，可根据需求选择合适的复杂度
  • 高性能 - 通过分片减少锁竞争，优化并发性能
  • 可扩展性 - 支持水平扩展，动态添加和移除节点
  • 韧性和可靠性 - 内置故障检测和自动恢复机制

核心功能​

内存缓存​

  • 高效的内存管理 - 优化的键值存储，支持各种数据类型
  • 自动过期机制 - 支持基于 TTL 的缓存项过期设置
  • 多种缓存清理策略 - 支持同步和异步清理过期项
  • 分片设计 - 使用多分片结构减少锁竞争，提升并发性能
  • 内存约束 - 可选的内存使用限制，支持 LRU 淘汰策略

分布式能力​

  • 一致性哈希分片 - 使用一致性哈希算法实现数据分片，减少节点变更时数据迁移
  • 虚拟节点 - 支持虚拟节点机制，提高数据分布均匀性
  • 节点权重 - 可为不同节点分配不同权重，适应异构环境
  • 远程缓存客户端 - 提供透明的远程节点访问能力
  • 可配置副本因子 - 支持数据多节点备份，提高可用性

集群管理​

  • 节点发现和成员管理 - 自动发现和维护集群成员列表
  • Gossip协议 - 基于 Gossip 的节点信息传播机制
  • 故障检测 - 健康检查和节点状态监控
  • 自动同步 - 节点间数据自动同步机制

多种集群架构​

  • 分片集群 - 数据通过一致性哈希分布在所有节点上
  • 主从架构 - 支持读写分离的主从复制模式
  • 选举机制 - 支持领导者选举，确保集群高可用

缓存一致性​

  • Cache-Aside 模式 - 经典的缓存旁路模式
  • Write-Through 模式 - 写透模式，同时更新数据源和缓存
  • 消息通知机制 - 基于消息队列的一致性策略，支持跨节点缓存失效

监控与可观测性​

  • 丰富的指标 - 内置命中率、延迟、内存使用率等指标
  • Prometheus 集成 - 支持 Prometheus 格式的指标导出
  • 事件通知 - 支持集群事件通知和自定义处理

API 和管理​

  • HTTP API - 提供 RESTful 管理接口
  • 健康检查 - 内置健康检查端点
  • 动态配置 - 支持运行时配置调整

使用场景​

FyerCache 适用于多种场景，包括但不限于：

  • 应用程序缓存 - 减轻数据库负载，提高应用响应速度
  • 会话存储 - 高性能的分布式会话管理
  • API 请求结果缓存 - 减少重复计算，提高 API 性能
  • 数据库查询缓存 - 缓存频繁访问的查询结果
  • 分布式锁和信号量 - 支持分布式协调
  • 高频计数和统计 - 实时统计和计数应用

架构概述​

FyerCache 采用模块化设计，主要由以下几个部分组成：

  1. 核心缓存引擎 - 提供基础的 Get/Set/Del 操作和内存管理
  2. 分片机制 - 实现并发友好的数据分片策略
  3. 一致性哈希 - 处理分布式环境中的数据分片和负载均衡
  4. 集群管理 - 负责节点发现、成员管理和故障检测
  5. 复制系统 - 处理主从复制和数据同步
  6. 监控子系统 - 收集和导出性能指标
