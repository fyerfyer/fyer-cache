Remote Cache Client

远程缓存客户端是 FyerCache 分布式系统中的关键组件，它允许应用程序透明地与远程缓存节点交互。远程缓存客户端实现了与本地缓存相同的接口，同时处理网络通信、错误重试和连接管理等复杂细节。

基本用法​

创建远程缓存客户端​

使用 NewRemoteCache 函数创建远程缓存客户端，提供目标节点 ID 和地址：

// 创建基本的远程缓存客户端
remoteCache := distributed.NewRemoteCache(
    "remote-node-1",     // 节点 ID
    "localhost:8080"     // 节点地址
)

执行基本操作​

远程缓存客户端实现了标准的 Cache 接口，可以直接使用 Get, Set 和 Del 方法：

ctx := context.Background()

// 设置缓存项
err := remoteCache.Set(ctx, "user:1001", "John Doe", 10*time.Minute)
if err != nil {
    // 处理错误
}

// 获取缓存项
value, err := remoteCache.Get(ctx, "user:1001")
if err != nil {
    // 处理错误
} else {
    fmt.Printf("User: %s\n", value)
}

// 删除缓存项
err = remoteCache.Del(ctx, "user:1001")
if err != nil {
    // 处理错误
}

高级配置​

RemoteCache 提供多个配置选项，可以根据需要定制客户端行为：

超时控制​

设置请求超时时间，防止请求长时间阻塞：

remoteCache := distributed.NewRemoteCache(
    "remote-node-1",
    "cache-server:8080",
    distributed.WithTimeout(500*time.Millisecond)  // 500ms 超时
)

重试策略​

配置网络错误的重试次数和重试间隔：

remoteCache := distributed.NewRemoteCache(
    "remote-node-1",
    "cache-server:8080",
    // 最多重试 3 次，每次间隔 100ms
    distributed.WithRetry(3, 100*time.Millisecond)
)

调试日志​

启用详细日志以便排查问题：

remoteCache := distributed.NewRemoteCache(
    "remote-node-1",
    "cache-server:8080",
    distributed.WithDebugLog(true)  // 启用调试日志
)

组合多个选项​

可以同时使用多个选项来精细配置远程缓存客户端：

remoteCache := distributed.NewRemoteCache(
    "remote-node-1",
    "cache-server:8080",
    distributed.WithTimeout(300*time.Millisecond),
    distributed.WithRetry(5, 50*time.Millisecond),
    distributed.WithDebugLog(true)
)

错误处理​

远程缓存客户端会遇到多种错误类型，包括网络错误、超时和服务器错误。以下是处理这些错误的常见模式：

value, err := remoteCache.Get(ctx, "user:1001")
if err != nil {
    if errors.Is(err, ferr.ErrKeyNotFound) {
        // 缓存未命中 - 从数据源加载
        value = loadFromDataSource("user:1001")
    } else if errors.Is(err, context.DeadlineExceeded) {
        // 超时错误 - 使用降级策略
        value = getDefaultValue()
        logTimeoutError(err)
    } else {
        // 其他错误 - 记录并降级
        value = getDefaultValue()
        logUnexpectedError(err)
    }
}

重试机制​

RemoteCache 内置了重试机制，可以自动重试临时性的网络错误：

  1. 设置适当的重试次数和重试间隔
  2. 捕获剩余错误并适当处理
  3. 考虑在应用层增加熔断器模式，防止级联失败
func doWithBackoff(ctx context.Context, key string) (any, error) {
    // 初始重试次数
    retries := 3
    
    // 指数退避
    backoff := 50 * time.Millisecond
    
    var lastErr error
    for i := 0; i <= retries; i++ {
        value, err := remoteCache.Get(ctx, key)
        if err == nil {
            return value, nil
        }
        
        lastErr = err
        
        if i < retries {
            // 等待一段时间后重试
            time.Sleep(backoff)
            // 增加下次等待时间
            backoff *= 2
        }
    }
    
    return nil, fmt.Errorf("all retries failed: %w", lastErr)
}

与节点分片缓存结合​

远程缓存客户端通常与节点分片缓存结合使用，实现完整的分布式缓存系统：

// 创建分片集群缓存
shardedCache := cache.NewNodeShardedCache()

// 为每个远程节点创建 RemoteCache 实例
remoteCache1 := distributed.NewRemoteCache("node1", "server1:8080")
remoteCache2 := distributed.NewRemoteCache("node2", "server2:8080")
remoteCache3 := distributed.NewRemoteCache("node3", "server3:8080")

// 添加到分片缓存，指定不同的权重
shardedCache.AddNode("node1", remoteCache1, 1)
shardedCache.AddNode("node2", remoteCache2, 2)
shardedCache.AddNode("node3", remoteCache3, 1)

// 使用分片缓存 - 自动路由到正确的节点
shardedCache.Set(ctx, "user:1001", userData, time.Hour)

内部实现细节​

RemoteCache 通过 HTTP 协议与远程缓存服务器通信：

  1. URL 路径：

    • GET /cache/get - 获取缓存项
    • POST /cache/set - 设置缓存项
    • DELETE /cache/delete - 删除缓存项

  2. 请求格式：使用 JSON 格式传递数据

    {
      "key": "user:1001",
      "value": "John Doe",
      "expiration": 600000000000  // 纳秒为单位
    }

  3. 响应格式：

    {
      "success": true,
      "message": "Cache key found",
      "value": "John Doe"
    }

  4. 错误处理：非 200 状态码或 JSON 解析错误会转换为适当的 Go 错误类型

本地操作限制​

由于 RemoteCache 客户端专门设计用于远程访问，以下操作不受支持：

// 不支持本地操作，总是返回错误
value, err := remoteCache.GetLocal(ctx, "key")  // 总是返回错误
err = remoteCache.SetLocal(ctx, "key", value, time.Minute)  // 总是返回错误
err = remoteCache.DelLocal(ctx, "key")  // 总是返回错误

这些操作专门为分布式缓存节点保留，在远程客户端上调用它们将返回 ErrOperationNotSupported 错误。

故障检测与处理​

RemoteCache 客户端内置的重试机制可以处理短暂的网络故障，但在生产环境中，您可能需要更复杂的故障处理：

// 创建具有短超时和多重试的客户端
remoteCache := distributed.NewRemoteCache(
    "node1",
    "server:8080",
    distributed.WithTimeout(200*time.Millisecond),
    distributed.WithRetry(3, 50*time.Millisecond)
)

// 使用自定义错误处理
func getWithFallback(ctx context.Context, key string) (any, error) {
    value, err := remoteCache.Get(ctx, key)
    if err != nil {
        recordError(err)  // 记录错误
        
        // 如果远程节点持续故障，考虑采取行动
        if isNodeFailing("node1") {
            triggerCircuitBreaker("node1")  // 触发熔断器
            return getFallbackValue(key)    // 降级处理
        }
        
        return nil, err
    }
    
    recordSuccess("node1")  // 记录成功
    return value, nil
}
